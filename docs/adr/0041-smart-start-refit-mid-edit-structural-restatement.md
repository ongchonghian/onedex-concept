# Smart Start refit: mid-edit structural restatement engine

> **Status:** accepted (2026-05-22)
> Builds on [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md) (structural-aware extraction contract that emits `refitSuggestions[]` as part of Layer 2's dual-output JSON), [ADR 0039](./0039-data-element-registration-admin-authored-single-page-flow.md) (the single-page registration flow refit operates inside, and the deliberation invariant in §80 that shapes refit's UI weight), [ADR 0038](./0038-data-element-governance-three-independent-layers.md) (the three-layer governance refit sits alongside), [ADR 0026](./0026-agreement-snapshot-immutability-schema-upgrade.md) (snapshot immutability — refit's cardinality-changing transforms must not silently mutate the published artefact and must preserve audit-defensibility post-restatement).

## Context

[ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s structural-aware extraction contract minimises class-3 schema issues at seed time, but real-world forms produce edge cases the structural classifier cannot fully prevent — radio-button clusters that VLM mis-saw as N independent boolean fields; table cells the LLM mis-emitted as flat per-cell properties despite the structural-region tag; repeated patterns that survived both the structural classifier and the name-pattern validator. The seed-time engine emits these as `refitSuggestions[]` self-audit candidates per §17, but Sarah has no UI to act on them. Without a deliberate review affordance, the residual class-3 problems either get hand-fixed (high friction; often abandoned) or get shipped as the wrong shape (footgun in published Elements).

The grilling session that produced [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md) also surfaced that **a "re-run the seed extraction" path is brutally asymmetric** — Sarah may have edited 30 fields by the time she notices a structural problem in field 31, and re-running discards all 30 edits. Without a path to repair structure *while preserving intermediate work*, the dominant outcome is hand-rebuild — which means most operators will ship bad structure rather than rebuild manually. The "preserve existing work" contract is what separates refit from re-running assist.

## Decision

Smart Start gains a third, mid-edit-only counterpart to **Smart Start seed** and **Smart Start assist**: **Smart Start refit** — a manually-triggered structural-restatement engine that operates against a draft already carrying Sarah's intermediate work and is contracted to preserve that work. Refit consumes the `refitSuggestions[]` emitted by [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s Layer 2 self-audit, plus autosave-debounced name-pattern detections, surfaces them in a side-drawer UI, and commits or rejects each through the standard audit chain.

Ten sub-decisions:

**1. Refit's scope is class-3 structural restatement only. Phase 1 is merge-direction only.** Three classes of mid-edit schema operation are now formally distinguished. **Value refinement** is intra-field, intra-type (add/remove enum options on an existing enum) — owned by the per-row sparkle button (Phase 2). **Type promotion** is intra-field, type-changing (`string` → `enum` on a single field) — owned by a separate explicit affordance with downstream-impact preview (Phase 2). **Structural restatement** is *region-shaped* (N fields ↔ 1 field) — owned by refit. Phase-1 refit is **merge-direction only** (N→1: collapsing booleans into enums, cells into `array<object>`, repeated patterns into `repeater-block`). The 1→N split direction requires deeper semantic reasoning the structural classifier cannot do from artefact + schema alone (knowing a generic string should be split into day/month/year is a semantic call, not a syntactic one) and is deferred to Phase 2.

**2. Refit suggestions are born from [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s Layer 2 self-audit, plus an autosave-debounced name-pattern scan, plus an opt-in manual VLM re-run.** Three trigger tiers matched to cost. **(a) Seed-time fold-in** — the Layer 2 LLM emits `refitSuggestions[]` as part of its mandatory dual-schema response per [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md). Zero additional inference cost; primary source of refit suggestions. **(b) Autosave-debounced regex scan** — runs locally on every autosave settle (`/^[a-z_]+_\d+_[a-z_]+$/i`); no inference call; updates the refit badge count if Sarah manually creates a flat numbered-key pattern mid-edit. **(c) Manual VLM re-run** — explicit button inside the refit drawer; fires a full structural re-detection against the original artefact bbox; throttled to one re-run per 60 seconds with a warning on rapid re-click (*"Last run was 8s ago — VLM extraction won't have new content"*). Cost-bounded per [ADR 0040 §67](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md).

**3. UI weight: button-with-count-badge in the Schema-tab header → right-side drawer with rich preview cards.** The Schema-tab header carries a *Structural review* button alongside "+ Add field" with a count badge ("3") when suggestions are queued, dark when none. Click opens a right-side drawer (~40% width; the field list stays visible at narrower width). Each suggestion is a card showing: the affected fields highlighted in the main panel via thin coloured outline as the card is hovered/focused; a before/after preview of the merged shape; cascade impact (validation rules that reference merged-away fields, sample-payload entries that would fail under the new shape); and Accept / Edit / Reject buttons. Rejected: always-on sidebar (steals real estate from the primary work), banner-on-detection (forces attention when refit is *available*, not *urgent* — class-3 patterns aren't deteriorating while Sarah works), modal overlay (hides the field rows being previewed, breaking the spatial mental model that makes structural restatement reviewable).

**4. Suggestion envelope is [ADR 0040 §32](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s universal shape with closed `kind` vocabulary.** Refit suggestions use the same envelope as every other Smart Start suggestion. `kind` discriminator is one of `structural-restatement.merge-to-{table|enum|array-object|likert}` — Phase 1 closed set, merge-direction only. `payload` carries `mergedFromFieldIds` (stable IDs from [register-element.js:81](../../portal-app/scripts/register-element.js:81), never raw names — survives Sarah renaming fields between seed time and drawer open) and `proposedField` (the merged shape). `sources[]` is non-negotiable per [ADR 0040 §32](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md): name-pattern matches with matched key list, bbox cluster regions, sibling-element citations. Justification prose lives inside `payload`, never replaces source citations.

**5. Cascade UX: a yellow rule-references banner with Apply rewrite / Skip / Edit-rule actions.** When Sarah accepts a structural restatement, the field-builder cross-checks every validation rule's `expression` string for references to `mergedFromFieldIds` (resolved to current field names). Any match surfaces a cascade banner *inside the drawer card before Apply commits*, listing each invalidated rule with three per-rule actions: **Apply rewrite** (best-effort substitution — e.g., `needs_inspection` becomes `recommended_action == 'inspection'` when merging four booleans into an enum), **Skip** (rule will fail at next evaluation; flagged with a chip on the Rules tab and an audit event `restatement-applied-with-orphan-rule`), **Edit-rule** (jumps to the Rules tab with the rule pre-selected for hand-rewrite). Accept-with-Skip is the *deliberate-leave-broken* path; the audit event captures Sarah's explicit acknowledgement so the regulatory query "did the operator know rule X was broken?" is answerable.

**6. Sticky rejection.** Once Sarah rejects a refit suggestion, it is stamped `dismissed` in the audit log and **not re-surfaced unless she explicitly clicks the *Re-run structural review* button inside the drawer**. The opposite (re-suggest until accepted) is a behavioural trap on slower clock per [ADR 0039 §80](./0039-data-element-registration-admin-authored-single-page-flow.md)'s deliberation invariant: Sarah rejects → suggestion comes back next session → she rejects again → eventually accept-clicks to make it stop, collapsing deliberation into accept-clicking. The explicit *Re-run* affordance preserves the deliberate-revisit path without continuous nagging.

**7. Audit event shape extends [ADR 0040 §40](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s closed vocabulary.** New event kinds: `suggestion-structural-restatement-emitted`, `-accepted`, `-edited`, `-rejected`, `-dismissed-persisted`, `restatement-applied-with-orphan-rule`, `refit-rerun-requested`. Payload carries `mergedFromFieldIds: string[]` for emitted/accepted events and `dismissedAt: ISO8601` for sticky-reject persistence. Element-version provenance per [ADR 0040 §40](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md) extends with an *optional* closed-schema sidecar on the surviving field: `mergedFrom: string[]` (the names of fields merged away). This preserves the post-merge regulatory query *"what fields became this?"* without bloating the immutable record — the audit log carries the full history, the Element version carries just enough to make sense of the published shape standalone.

**8. Capability gate: same as the rest of the registration flow.** [`canManageDataElements`](../../CONTEXT.md) (already governs Smart Start assist invocation per [ADR 0040 §52](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)) extends to refit invocation. No new capability needed; refit is part of the same authoring act. Future split into a dedicated `canUseSmartStartRefit` is non-breaking but unnecessary for v1 (no operational segmentation justifies it).

**9. Refit availability is gated on artefact retention.** The *Structural review* button is enabled when the original on-ramp artefact (PDF/Form/image) is still attached to the draft. Drafts seeded from non-artefact on-ramps (Plain English NL, Sample CSV/JSON, Fork) have no original artefact to re-examine and surface only the autosave-debounced name-pattern detections — the manual VLM re-run button is disabled with a tooltip *"Refit needs the original PDF/Form to detect structural patterns visually."*. Phase 2 may add structural detection for Plain English / Sample on-ramps if their seed shapes prove to generate enough class-3 problems to warrant it.

**10. Phase 1 scope deliberately conservative.** Phase 1 ships: the refit drawer + button-with-badge UI; consumption of Layer 2 `refitSuggestions[]` (the primary source); the autosave name-pattern scan; the cascade UX for rule references; sticky rejection; the audit-log event shape; throttled manual VLM re-run for artefact-bearing on-ramps. **Phase 2 deferrals:** structural detection for non-artefact on-ramps; split-direction restatements (1→N); per-row sparkle button for value-refinement (class-a); type-promotion modal (class-b); refit-suggestion bulk-review UX if usage data shows N>5 suggestions per registration becoming common; refit-suggestion-history view (revisit an Element to see the full restatement journey).

## Consequences

**Architectural:**
- The `regDraft` state shape gains a `refitSuggestions` substructure (mirroring the existing `assist` substructure from [ADR 0040](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)) — each entry uses [ADR 0040 §32](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s universal envelope.
- The Element-version `source.*` provenance gains an optional `mergedFrom: string[]` field on surviving-after-restatement fields. Like other provenance fields, this is a one-way commit per [ADR 0040 §62](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md) — once shipped, removing it is a breaking change.
- The autosave-debounced name-pattern scan runs in the existing autosave path ([`regScheduleAutosave`](../../portal-app/scripts/register-element.js:203)) — cheap regex, no inference call, no new dependencies.
- The Schema-tab header gains a *Structural review* button next to "+ Add field". Implementation lives alongside the field-builder layout code.

**Process / operational:**
- Phase-1 LLM cost is unchanged from [ADR 0040 §16](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md) — refit consumes seed-time suggestions; the manual VLM re-run is opt-in and throttled per ADR 0040's cost-bounded principle.
- Audit-log volume grows by the new event kinds; queryability remains the same.

**Forward-compatibility:**
- Split-direction restatements (1→N) extend the `kind` vocabulary to `structural-restatement.split-into-{enum-pair|object-fields|array-elements}` without changing the envelope.
- The per-row sparkle button (class-a value refinement) and type-promotion modal (class-b) inherit the drawer-style preview pattern; the drawer UI becomes reusable infrastructure across all three mid-edit assist classes.

## Considered alternatives

- **Active AST folding of detected name-patterns as a deterministic post-LLM transform** (silently fold `item_1_qty`/`item_2_qty` into `array<object>`). Rejected: violates [ADR 0039 §80](./0039-data-element-registration-admin-authored-single-page-flow.md)'s deliberation invariant — *"the auto-suggest pattern in a high-stakes safety gradient quietly converts the admin choice from deliberation to accept-clicking"* — and breaks the audit chain (Sarah sees folded output with no signal a fold happened; a regulator cannot later distinguish "extracted as `array<object>`" from "extracted flat, secretly folded"). Replaced by the bounded-retry validator (at LLM output time) + refit-suggestion-emission (for Sarah's deliberation) pattern in [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md).
- **"Re-run the seed extraction" as the only path to fix class-3 problems.** Asymmetric: Sarah's intermediate edits are nuked. Dominant operator outcome under that design is "ship bad structure rather than redo 30 edits." Refit's "preserve existing work" contract is the explicit corrective.
- **Auto-banner on detection (no manual trigger).** Forces attention when refit is *available*, not *urgent* — class-3 patterns aren't deteriorating while Sarah works. Trains banner blindness over a session. Replaced by button-with-count-badge (passive presence, not active interruption).
- **Continual re-suggestion of rejected suggestions until accepted.** Trains accept-clicking to make the prompt stop. Trades short-term acceptance rates for long-term collapse of deliberation. Sticky rejection is the correct shape; explicit *Re-run* gives the deliberate-revisit path.
- **Refit as a separate page / modal away from the field-builder.** A modal hides the affected rows being previewed; a separate page breaks the spatial mental model. Drawer keeps both visible.
- **Bulk "accept all" affordance for refit suggestions.** Same anti-pattern as the seed-time "bulk accept" rejected in [ADR 0040 §50](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md): collapses deliberation across N decisions in one click. Not Phase 1; revisit only with usage data showing genuine fatigue from N>5 suggestions per registration.
- **A separate LLM call dedicated to structural classification at seed time.** Rejected for Phase 1: doubles inference cost and stacks latency. Folded into [ADR 0040 §17](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md)'s Layer 2 dual-output instead, since the LLM already holds the full context (Layer 1 regions + bounding boxes + the schema it just engineered) and can self-audit without a second pass. Phase 2 may split if self-audit proves too weak.

## Future work (Phase 2 and beyond)

- **Split-direction restatements (1→N).** Requires deeper semantic reasoning to distinguish *"this string field should be three typed sub-fields"* from *"this string field is correct"*. Likely requires the deferred per-DEX wiki grounding source per [ADR 0040's "Per-DEX app-maintained LLM-friendly wiki"](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md) future-work entry.
- **Per-row sparkle button for value-refinement (class-a).** Cheap inline assist on individual fields — *"suggest more enum options"*, *"propose validation pattern"*. Reuses the drawer UI for preview.
- **Type-promotion modal (class-b)** with downstream-impact preview (rules invalidated, payload entries that would fail under the new type). Higher friction matched to higher stakes per the same friction-asymmetry principle that justifies the cascade UX in sub-decision 5.
- **Refit for non-artefact on-ramps** (Plain English / Sample / Fork) if usage data shows class-3 problems originate there too. Structural detection without a visual artefact requires different signals (sample-payload statistical clustering, sibling-element shape priors).
- **Refit-suggestion-history view.** Sarah revisiting an Element version sees the full restatement journey — which suggestions were emitted at seed time, accepted/rejected/dismissed, and when — analogous to the Phase 2 audit-log UI surface deferred in [ADR 0040 §72](./0040-smart-start-assist-grounded-cross-tab-suggestion-engine.md).
